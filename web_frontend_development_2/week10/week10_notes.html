MDN: Validating Forms

  • Here is some example code for client-side validation
  • there are a number of form controls available directly through HTML5, such as pattern, required, minlength, maxlength, min, max, and type
  • css pseudo-classes such as :valid or :invalid help the user to know what they need to fix
  • if you want to have more control over client-side validation and error messages, use Javascript
  • the constraint validation API is very useful; one great thing about it is that you can set custom error messages that are very explicit about what didn't validate so that the user doesn't have to guess what went wrong

MDN: Using Fetch

  • here is a fetch request example:
    fetch('http://example.com/movies.json')
    .then(response => response.json())
    .then(data => console.log(data));
