Week 1 Notes

Doing Stuff with Web Things

Here are my takeaways from the reading:

  • I like the analogy that Javascript variables are like little labeled boxes to hold things in
  • to get an input, use .value
  • to output something, use .innerHTML
  • localStorage can be used to save things for later on the same browser
  • localStorage uses key/value pairs
  • key/value pairs remind me of the analogy used for Javascript variables, only in this case the label on the box = the key and what's in the box = the value
  • when you ask localStorage to retrieve a value(the stuff in the box), it only needs to know the key(the label on the box)

Mobile First Design

The majority of people use mobile devices and not desktops to view webpages. Get with the times and always design for mobile before you even worry about designing for larger screens. Here are some takeaways:

  • low attentions spans/high laziness = make it easy
  • keep reachability in mind & put the most important things closer to the bottom of the screen
  • speed is very very important (cut down on images, improve reachability to change speed perception)
  • make it obvious (the word "menu," for example, is better than the hamburger button icon)
  • designing with mobile in mind forces you to prioritize, cut out unnecessary junk, and simplify (this is especially important with forms)
  • can use placeholder text in input fields rather than labels in order to save space
