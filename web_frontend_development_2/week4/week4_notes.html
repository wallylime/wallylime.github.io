Week 4 Notes

Ch. 8: Forms

  • using document.forms will return a list of all of the forms in the document, so you have to use either index notation or dot notation with the form's name to find what you're looking for
    const formVariable = document.forms[indexNumber] or const formVariable = document.forms.formNameHere or const formVariable = document.forms["formNameHere"]
  • checkbox inputs need to all have the same name property to be grouped together
  • checkbox inputs are considered checked when their boulean value is true and unchecked when false
  • radio buttons are like the checkbox input except that only one option can be selected
  • the submit button can be disabled if there are invalid form inputs

Ch. 12: Object-Oriented Programming

  • class declarations and constructor functions are both capitalized
  • class declarations are better than constructor functions
  • static methods are available only to the class and not instances of the class
  • properties and methods can be added to a class using the prototype property
    Class.prototype.newPropertyName = "New Property Value";
    Class.prototype.newMethodName = function () {return statement;}
  • prototype properties will be shared by all the objects that share the same class; they are not customizable and are not considered the object's "own" property
  • here's the difference between private and public properties/methods
  • one class can inherit from another using extends
  • getter and setter methods are especially used when a property value is dependent on other property values
  • getter and setter methods allow more control over property assignment
  • instead of using classes, you can create an exact copy of an object with .create
    const ObjectYouWantToCopy = {property1: 1, property2: 2, etc.};
    const newObjectName = Object.create(ObjectYouWantToCopy);
  • if you change the prototype object, it will affect all of the other objects that use it as their prototype
  • the instanceof operator only works on objects built with constructor functions
  • object properties can be given through assignment like this:
    const a = {}; const b = { name: 'JavaScript' }; Object.assign(a,b) << { name: 'JavaScript' } a.name << 'JavaScript'
  • the problem with assignment is that arrays and functions that are stored as properties only REFERENCE the original; they don't store their own copy of the property value
  • mixins are functions that can be used to assign "deep" object properties (properties that aren't just REFERENCING)
  • the variable "this" can get messy and refer to the wrong object
  • there are several methods for pinpointing what "this" is referring to, including using that = this
  • try to make objects like small, reusable blocks that can be used to build bigger things

Modular JavaScript

  • you can organize different parts of Javascript into files called modules
  • the keywords "export" and "import" are used to actually access the code from another file
