Ch. 10: Testing and Debugging

  • warnings won't immediately cause the program to crash, but can come back to bite you later
  • code in strict mode which will follow coding best practices and throw errors instead of warnings
  • to code in strict mode, just write the string "use strict" at the top of the file or within a function that contains all of your code:
    (function() { 'use strict'; // All your code would go inside this function }());
  • the main reason for using "use strict" within a function instead of at the top of the file is that if you are collaborating on code, someone else might not have used strict mode and if you put it at the top of the file, strict mode will be imposed on their code as well
  • linting tools are even more strict than "use strict"
  • using feature detection inside of an if statement helps the program to "fail gracefully" if the user's browser doesn't support the feature in question
  • use the console rather than the alert box for debugging purposes
  • using the debugger tool is even more efficient than using console.log
  • info on how to use debugger in Chrome
  • you can use the keyword throw to create your own errors if a certain situation occurs
  • try, catch, and finally can be used to deal with errors
  • "try" is for the code that you want to run but might cause an error
  • "catch" is for the code to run if the code in "try" does cause an error
  • "finally" is the code that should be run regardless of whether the code from "try" or the code from "catch" was used
  • TDD stands for test-driven development
  • start with test cases before actually writing any code
  • JEST is a testing framework that works on files ending with .test.js
