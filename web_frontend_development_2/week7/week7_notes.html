Ch. 11: Further Functions

  • a function's length property returns the number of parameters that it has
  • when calling a function, it's first argument is always the value of "this"; other arguments can follow
  • memoization is basically saving the results of an operation as a property of the function so that if the function is called again with the same argument, it can get the answer that has been stored as a property without having to calculate it again
  • immediately invoked function expressions (IIFE's) sound like a bad idea to me; the only use case that seems to make sense is coding in strict mode for your own code without affecting others' code
  • functions can redefine themselves by using an anonymous function
  • recursive functions keep going until a condition is met
  • here's a video explaining the Javascript event loop
  • Javascript can only do one thing at a time
  • the results of an asynchronous operation is called a promise
  • promises are either resolved or rejected
  • the promise is "pending" while it's waiting to be resolved or rejected
  • the ".then" method says what to do next after the results of a promise are in
  • you can chain promises together in a sequential way using the ".then" method
  • when functions are nested inside of each other, the inner functions have access to the variables that are scoped for the outer function
  • Here is a confusing part of the textbook about closures
  • generators are called like this:
    function* exampleGenerator() { // code for the generator goes here }
  • generators use the keyword "yield" instead of "return"; this allows the yield result to be available the next time the generator is used
  • ".next" method is used to pass through the generator again
  • here's a list of criteria for pure functions
  • a programming style that uses single-purpose pure functions as building blocks for other functions is called functional programming

Ch. 13: AJAX

  • AJAX = asynchronous Javascript and XML
  • JSON is used more than XML now
  • here's an example of using the fetch() method:
    fetch('https://example.com/data')
    .then( // code that handles the response )
    .catch( // code that runs if the server returns an error )
  • rather than creating a new Request object, you can put it directly in the fetch request as one of the arguments, like this:
    fetch('https://example.com/data', {
    method: 'GET',
    mode: 'cors',
    redirect: 'follow',
    cache: 'no-cache'
    })
    .then( // do something with the response )
    .catch( // handle any errors)
  • the FormData interface simplifies submitting form data
