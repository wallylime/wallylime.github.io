Advanced CSS Notes

Selectors

  • descendant combinator = A B (this will select B anywhere it is nested within A)
  • child combinator = A > B (this will select B only if it is a direct child of A)
  • adjacent sibling combinator = A + B (this will select B only if it is immediately following A as children of the same parent element
  • general sibling combinator = A ~ B (this will select B if it is following A (does not need to be immediate) and they are both children of the same parent element
  • there are various attribute selectors, most of which sound lame
  • pseudo-elements that I like:
    • ::first-letter (selects the first letter of the element for styling in an adorable way)
    • ::first-line (selects the first line of the element for styling)
    • ::selection (can change how it looks when the user highlights something)
  • pseudo-classes that I like:
    • :target (use to style jump links--really cool example of using this to create tabs starting on pg. 40)
    • :not(argument) (returns elements without whatever you've put in paranthesis)
    • :first-child, :last-child
    • :nth-child(argument), :nth-last-child(argument) (the argument can be even, odd, the child number, or a step interval --even and odd can be especially useful in styling tables)
    • :required (useful for styling required fields in a form)
    • :valid and :invalid (used on forms)
  • pseudo-classes can be chained together
  • keep specificity low
  • calculate specificity by the number of id selectors, class/pseudo-class/attribute selectors, and type/pseudo-element selectors

Variables in SASS

  • variables are declared using $
  • to update a global variable to a new value, use !global
  • to add a value if it has not already been set, use !default
  • use #{$variableName} to keep from having to do a lengthy concatenation
  • mixins remind me of a high-bred between variables and functions
  • declare a mixin with @mixin and then the name of it; use a mixin with @include and the name of the mixin
  • a function returns a value; a mixin outputs CSS code
  • using CSS native custom properties allows you to manipulate them with JavaScript
  • CSS custom properties cascade
  • the major drawback of CSS custom properties vs. using a preprocessor is that it doesn't have very good browser support yet
